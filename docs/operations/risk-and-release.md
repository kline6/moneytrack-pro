# 风险、质量与上线检查

## 风险清单

### 高风险
1. **金额精度错误**
   - 表现：分/元转换、浮点计算导致账本对不上
   - 控制：数据库统一存储最小单位，禁止浮点金额进核心模型
2. **离线重复提交**
   - 表现：断网恢复后同一笔账被重复入库
   - 控制：`clientTxnId` 幂等校验
3. **预算统计不一致**
   - 表现：删改账单后预算页和首页数据不一致
   - 控制：预算统计走统一聚合查询，关键汇总基于事务一致读
4. **首版过度设计**
   - 表现：模型过早支持多端、多年度预算、复杂冲突仲裁
   - 控制：严格执行 v1 范围
5. **用户输入污染**
   - 表现：搜索词、分类名、备注注入异常字符
   - 控制：Zod 校验 + 白名单字符集 + 长度限制 + 参数清洗

### 中风险
1. 列表筛选组合导致慢查询
2. 附件上传类型与大小失控
3. 前端离线缓存膨胀
4. 多页面重复请求首页汇总接口
5. 主题系统和 Android 异形屏适配差异

### 低风险
1. 分类图标缺失
2. 图表空数据展示
3. 设置页偏好存储不一致
4. 导出字段顺序问题
